### Citation discipline

**F5 (P2) — Knowledge files contain bare line-number citations against `_conventions.md` rule.**

Per `_conventions.md` "Citation discipline": "Knowledge files should cite by **symbol name + file**, not by line number." Today's knowledge updates have several violations:

- `qa-reviewer/knowledge/known-coverage-gaps.md`: `useCanvasViewport.test.ts:43-55, 250-256` — bare line ranges, no symbol. `AutomatonCanvas.tsx:227-232`, `useFileSession.ts:107-115` — bare line ranges; the surrounding text names the symbol so this is the "acceptable hint" pattern, but the citation form is mixed.
- `qa-reviewer/knowledge/test-coverage-map.md`: `useKeyboardScope.test.ts (line 161)` — bare line, no test name.
- `typescript-reviewer/knowledge/optional-prop-policy.md`: `parseAutomataFile line 110`, `useFileSession.ts:122` — symbol named in the first, bare line in the second.

The convention permits bare lines in journals (where `diff-hash` documents the snapshot), but knowledge files mutate. Recommend a sweep: convert all bare line numbers in knowledge files to `symbol + file` (with line as parenthetical hint).
